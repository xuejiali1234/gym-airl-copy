According to a document from (2026-04-19)，最新这轮 probe 已经把一件事说明白了：`n_disc=5` 这条线里，最优点是 `D5_S08`，也就是 `N_DISC_UPDATES=5 + SAFETY_REGULATOR_COEFF=0.08`；但它仍然没有超过你当前全局最优旧基线。`P300_D5_S08` 在 300 轮做到 `merge=0.75 / endpoint=1.00 / safety=0.75 / collision=0.25 / env_return=1.625 / dense=38.46`，明显好于 `S09/S10`；但旧基线 `20260418_224355` 和 `20260418_180202` 仍然更强，分别到 `env_return=1.80/1.775`，同时安全率更高、碰撞率更低。 

把三轮 probe 串起来看，逻辑是这样的。第一轮 200 轮里，`P200_C` 一度让你觉得 `sreg=0.10` 更好；第二轮 200 轮里，`P200_D5` 又证明了把判别器更新从 6 降到 5，确实能让对抗平衡更健康，`reward_gap` 和 `value_loss` 都明显好于 control；第三轮 300 轮进一步说明，在 `D5` 分支里，最优 `sreg` 会下移到 `0.08`，继续推到 `0.09/0.10` 只会伤后期表现。换句话说，`n_disc` 和 `sreg` 是耦合的，200 轮时看到的“高 sreg 更好”不能直接外推到 300 轮。  

所以，下一步我不建议再做“往上推安全正则”的散扫，而是做一个**最小但信息量最大的局部窄扫**。核心问题现在不是“0.09 还是 0.10”，而是：**全局最优到底还在旧的 `D6_S06` 家族里，还是 `D5` 这条更稳的线在更低 `sreg` 区间里还有没被验证的长程潜力。** 目前最缺的数据其实不是 `D5_S09/S10`，而是 **`D5_S06` 的 300 轮结果**，以及 **`D6_S07` 这种紧邻旧基线的轻微上探**。 

我建议下一轮直接跑这 3 组，其他参数全部固定不动：

1. `P300_D6_S07`：`ppo6 / glr=8e-5 / disc_lr=5e-5 / n_disc=6 / sreg=0.07 / ent=0.005 / reward_norm=off`
2. `P300_D5_S06`：`ppo6 / glr=8e-5 / disc_lr=5e-5 / n_disc=5 / sreg=0.06 / ent=0.005 / reward_norm=off`
3. `P300_D5_S07`：`ppo6 / glr=8e-5 / disc_lr=5e-5 / n_disc=5 / sreg=0.07 / ent=0.005 / reward_norm=off`

这 3 组里，`D6_S07` 是“最有希望直接超过全局旧基线”的那一组，因为它留在当前最强家族上，只做很小的安全权重上探；`D5_S06` 是“最该补的空白点”，因为它 200 轮时已经表现不错，但你从来没让它完整跑到 300；`D5_S07` 则是 `D5_S06` 和 `D5_S08` 之间最合理的中点。算力只够跑 1 组时，先跑 `D6_S07`；算力够 2 组时，再加 `D5_S06`。  

反过来，下面这些方向现在可以先停掉：`sreg >= 0.09`、`reward normalization`、`disc_lr=4e-5`、`glr=7e-5`、`ent=0.003`。前两轮 probe 已经足够说明，它们要么没有带来收益，要么会把策略推向“更保守但不更好”，或者直接让长期表现变差。  

挑模型时也别再用“奖励曲线像不像经典 RL 曲线”当主标准。你现在 `Policy_Return_Components_Norm100` 的脚本，本质上只是把稀疏环境奖励 `merge/endpoint/safety` 按 `25/25/50` 线性拆开，它天然就是 success-component 曲线，不会长成 MuJoCo 那种 dense return。论文里真正该看的是效率、安全、舒适这三类指标，比如 merging success、collision、TTC/headway、jerk，而不是某条 reward 曲线是否好看。  

下一轮具体选 checkpoint，我建议只看每 20 轮一次的 full-eval 点，用一个简单的综合分数来排：
`score = env_return + 0.5*safety + 0.25*merge - 0.5*collision - 0.03*jerk`
仍然只取 **best checkpoint**，不要默认用 final。你现在的目标不是把曲线“修成 RL 样子”，而是先回答一个更重要的问题：**300 轮下，`D5` 的稳定性到底能不能转化成比旧 `D6_S06` 更高的最终任务表现。**
